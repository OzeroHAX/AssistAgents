# 智能体及其使用场景

## 主要智能体

- `build-dev` - 代码实现、修复和逐步验证。
- `build-planner` - 只读实现规划。
- `test` - 测试以及测试报告和缺陷报告产物。
- `review` - 只读代码审查和风险评估。
- `doc` - 允许范围内的用户/运维文档。
- `project` - 项目管理智能体：规划、分解和维护项目工件。
- `ask` - 快速提问和研究，不进行编辑。

## 子智能体

系统还包含用于委托的专业子智能体：

- 代码研究；
- 网络研究；
- 分解。

主要智能体在需要专注分析时会调用这些子智能体。

## 智能体选择指南

- 需要在编辑前制定计划 -> `build-planner`。
- 需要代码更改 -> `build-dev`。
- 需要验证和报告 -> `test`。
- 需要独立的质量/风险审查 -> `review`。
- 需要用户面向的文档 -> `doc`。
- 需要项目级工作（目标、范围、PRD、架构、史诗、用例、任务）-> `project`。
- 需要快速答案或研究 -> `ask`。
