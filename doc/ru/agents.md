# Агенты и когда какой использовать

## Основные агенты

- `build/dev` - реализация изменений в коде, фиксы, пошаговая верификация.
- `build/planner` - read-only планирование реализации.
- `test` - тестирование и оформление тестовых отчетов и баг-репортов.
- `review` - read-only code review и оценка рисков.
- `doc` - пользовательская и операционная документация в разрешенной docs-области.
- `project` - агент управления проектом: планирование, декомпозиция и ведение проектных артефактов.
- `ask` - быстрые вопросы и исследования без правок.

## Подагенты

Система также содержит специализированные подагенты для делегирования:

- code research;
- web research;
- decomposition.

Их обычно вызывает основной агент, когда нужен узконаправленный анализ.

## Практика выбора агента

- Нужен план перед правками -> `build/planner`.
- Нужны правки в коде -> `build/dev`.
- Нужны проверки и отчет -> `test`.
- Нужна независимая проверка качества -> `review`.
- Нужна пользовательская документация -> `doc`.
- Нужна работа с проектом как системой (цели, scope, PRD, архитектура, эпики, use-cases, задачи) -> `project`.
- Нужен быстрый ответ или ресерч -> `ask`.
