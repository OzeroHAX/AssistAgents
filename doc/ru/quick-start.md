# Быстрый старт

Этот сценарий помогает начать работать с AssistAgents сразу после установки.

## 1) Установите пакет

```bash
npx -g @ozerohax/assistagents@latest
```

## 2) Убедитесь, что структура установилась

Ожидаемые каталоги в `~/.opencode`:

- `agents`
- `skills`
- `keys`
- `opencode.jsonc`

## 3) Сгенерируйте project-local coder skills (опционально)

Если в проекте нужны локальные правила кодинга под ваш стек, используйте слеш-команду:

Запускайте ее через агента `build-dev`.

```text
/init-agent-assist-code
```

Что делает команда:

- создает или обновляет skills только в `.opencode/skills/coder/` внутри текущего репозитория;
- определяет языки автоматически по коду и конфигам, если аргументы не переданы;
- поддерживает явный список языков через `$ARGUMENTS` (comma-separated), например:

```text
/init-agent-assist-code typescript,csharp
```

Когда использовать:

- сразу после установки/обновления AssistAgents для нового проекта;
- после заметного изменения стека (новый язык, фреймворк, тестовый раннер, линтер);
- когда хотите синхронизировать локальные coder skills с текущим состоянием репозитория.

## 4) Используйте рекомендуемый рабочий цикл

Короткие итерации работают надежнее, чем один большой запрос:

1. `build-planner` - собрать минимальный и проверяемый план.
2. `build-dev` - реализовывать по шагам.
3. `test` - проверить поведение и зафиксировать результаты.
4. `review` - прогнать quality/risk-проверку.
5. Повторять цикл до выполнения критериев готовности.

## 5) Минимальный стартовый сценарий

Пример первой итерации:

1. Дайте `build-planner` задачу на 1 конкретное изменение.
2. Передайте план в `build-dev` и попросите выполнить шаги с верификацией.
3. Запустите `test` для подтверждения результата.
4. Запустите `review`, чтобы проверить риски и качество.

## 6) Когда использовать `ask`

`ask` удобен для быстрых вопросов и read-only исследования контекста без выполнения изменений.

## 7) Когда подключать `project`

`project` используйте, когда нужна именно работа с проектом, а не с отдельным изменением кода:

- формализация требований и границ (scope);
- ведение PRD, архитектурных решений и use-cases;
- декомпозиция в эпики/задачи;
- актуализация проектного статуса и артефактов перед реализацией.
